# attrs-admin

归因数据管理系统前端管理台，基于 `@umijs/max`、`Ant Design` 和 `Ant Design Pro Components` 构建，用于统一管理应用、事件、订阅、通知、系统配置以及权限体系，并提供订阅营收分析看板。

## 项目简介

本项目是 `GodHelp` 归因后台的管理端，主要面向运营、技术支持和后台管理人员，提供：

- 登录鉴权与 JWT 会话管理
- 基于权限码的按钮级访问控制
- 应用、事件、订阅、通知等业务数据管理
- 用户、角色、权限、组织、账号等系统管理能力
- 订阅分析、收入趋势、渠道分析、LTV 等数据看板
- 中英文国际化支持

## 技术栈

- `React 18`
- `TypeScript`
- `@umijs/max`

## 功能模块

### 1. 登录与鉴权

- 登录页支持用户名、密码登录
- 支持验证码获取与刷新
- 登录成功后将 JWT 保存到本地
- 应用启动时自动解析 JWT，并拉取当前用户权限码
- 当接口返回 `401` 时自动清理登录态并跳转到登录页

### 2. 数据大盘

数据大盘聚合了多个维度的经营分析能力，包括：

- 应用筛选与时间范围筛选
- 核心指标统计
  - 今日订阅数
  - 本月订阅数
  - 本月订阅金额
  - 累计订阅金额
- 订阅/收入日趋势分析
- 收入汇总分析
- 按产品收入分布
- 渠道总览与广告活动分析
- 国家收入分析
- 渠道 LTV 分析
- Cohort LTV 分析

### 3. 系统管理

#### 用户管理

- 用户列表查询
- 用户详情查看
- 创建、编辑、删除用户

#### 角色管理

- 角色列表管理

- 为角色分配权限

#### 权限管理

- 权限模块、路由、层级配置

#### 组织管理

- 公司/组织信息管理
- 支持公司下拉选择能力，供其他业务模块复用

#### 账号管理

- 渠道账号/业务账号管理

## 权限控制说明

项目使用“权限码”做前端按钮级控制：

- 启动时通过 `/admin/permission/user-permissions` 拉取当前用户权限码列表
- 页面中的操作按钮通过 `AccessButton` 组件按权限码控制是否展示
- 用户名为 `admin` 时默认拥有全部权限
- 典型权限码示例：`user:create`、`user:update`、`role:delete`、`settings:update`

这意味着：

- 路由是否可访问主要依赖登录态
- 按钮、操作入口是否显示依赖后端返回的权限码集合

## 环境变量

项目当前使用到的环境变量：

| 变量名 | 说明 | 示例 |
| --- | --- | --- |
| `PORT` | 本地开发端口 | `8805` |
| `API_TARGET` | 后端服务地址，开发环境通过代理转发 `/admin` 请求 | `http://localhost:8802` |

示例 `.env`：

```bash
PORT=8805
API_TARGET=http://localhost:8802
```

## 接口与代理约定

- 前端统一请求 `/admin/**`
- 开发环境下通过 `.umirc.ts` 中的 `proxy` 转发到 `API_TARGET`
- 请求拦截器会自动附带：
  - `Authorization: Bearer <token>`
  - `Accept-Language`
- 响应拦截器默认约定后端返回结构为：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

约定行为：

- `code === 0`：请求成功，前端会直接解包 `data`
- `code === 401`：登录失效，自动跳转登录页
- 其他非 `0` 状态码：统一弹出错误提示

## 本地开发

### 环境要求

建议使用：

- `Node.js 18+`
- `npm 9+`

### 安装依赖

```bash
npm install
```

> 安装完成后会自动执行 `max setup`。

### 启动开发环境

```bash
npm run dev
```

默认访问地址：`http://localhost:8805`

### 运行测试

```bash
npm run test
```

### 构建生产包

```bash
npm run build
```

构建产物默认输出到 `dist/` 目录。

当前内置语言：

- 简体中文：`src/locales/zh-CN.ts`
- 英文：`src/locales/en-US.ts`

请求头中会自动携带当前语言，用于服务端按语言返回文案或错误信息。

## 部署说明

部署前请确认：

- 已正确配置生产环境 `API_TARGET`
- 后端已开放对应 `/admin/**` 接口
- 静态资源部署后可正确回退到前端入口路由

典型流程：

1. 配置环境变量
2. 执行 `npm run build`
3. 部署 `dist/` 静态文件到 Web 服务器
